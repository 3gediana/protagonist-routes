# 案例 01：振幅卡在 0.004–0.008，撞不破 0.009 天花板

> PE 的核心瓶颈。示范了"用预判器先确认天花板是否真的撞不破，再决定要不要换协同演化策略"，而不是盲目调 NEAT 超参。

---

## 1. 问题现象（精确陈述）

- 关注量：振幅 amplitude（行为/生态指标随代数的振荡幅度）。
- 现象：多个 seed 的振幅在 **0.004–0.008** 之间徘徊，**没有任何一个 seed 接近 0.009 的天花板**。

## 2. 确定原因的方法

### 2.1 先用早代预判器判断"是不是所有 seed 注定都撞不破"
不等每个 seed 跑满 80 代，用 **早代预判器**（见 `02` 篇）从 gen0–15 的特征预测 gen79 的结果。在 3 个 sC 完整 80 代跑上做 LOSO（留一交叉验证）标定，MAE ≈ 0.0006。

预判结论：**没有任何 seed 的预测终值接近 0.009** → 这不是"个别 seed 运气差"，而是**当前演化配置下的系统性上限**。这一步避免了"换个随机种子再赌一把"的盲目重跑。

### 2.2 读 harmony 指标的协同结构
7 个 harmony 指标不是独立的——振幅的天花板来自指标之间的**相互制约**（拉高一个会压低另一个）。单独优化任一指标（A 或 E）都会被其它指标拽回 → 振幅上不去。

## 3. 根因（当前最佳判断）

振幅平台是**协同约束**造成的：在现有 NEAT 配置下，种群无法同时把相互制约的多个 harmony 指标推高 → 振幅被锁在 0.008 以下。需要的是**协同演化策略**（co-tune 多个指标，例如 A+E 联合），而不是单指标加压或换种子。

## 4. 调试 / 实验设计

- 起 3 个 **sD_warm** seed（从 gen80 热 resume），目标 gen160，尝试 co-tune。
- ⚠️ 重要约束：振幅只在 tail（gen140–160）写盘，所以这几个长跑**要推进到 gen140+ 才能读到真振幅**——这就是为什么"分钟级筛"在 PE 这种长跑上不适用（见 `03` 篇）。

## 5. 跑测试验证（判据）

- 等 3 个 sD_warm seed 到 gen140+，读 tail 振幅；
- 判据：是否有 seed 的 tail 振幅突破 0.009；
- 若突破 → 锁定该 co-tune 配置；若仍 ≤0.008 → 说明 co-tune 方向/力度不对，按预判器重新选下一组指标组合。

> 【当前状态：3 个 sD_warm seed 在 gen85–99 区间推进，尚未到 tail，振幅待读。】

## 6. 一句话给接手模型

**振幅撞不破 0.009 不是随机种子运气问题——早代预判器（LOSO 标定 MAE~0.0006）已证明是系统性上限，根因是 7 个 harmony 指标的协同制约。解法方向是 co-tune 多指标（如 A+E），不是单指标加压。验证必须等长跑到 gen140+（振幅只在 tail 20 代写盘）。**
