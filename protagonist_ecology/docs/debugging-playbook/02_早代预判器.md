# 案例 02：早代多行为预判器（怎么造、怎么标定、什么时候不能信）

> PE 版的"分钟级读信号"工具。重点不是它预测得多准，而是**怎么验证它可信，以及它的有效边界在哪**。
> 对应文件：`pe_early_predictor.py`、`pe_probe.py`、`PE_EARLY_PREDICTOR.md`（commit f1e5bff）。

---

## 1. 动机 / 问题

- PE 一个 seed 跑满 80 代很慢；若每个配置都跑满才看终值振幅，迭代极慢。
- 需要：从**早代（gen0–15）**的可观测特征，预测**终代（gen79）**的多个行为/和谐指标，从而几分钟内筛掉没希望的配置。

## 2. 怎么造

- 输入特征：gen0–15 的早代统计（多行为：spears/fires/worksite/chop/craft 等的早期斜率与水平）。
- 输出：gen79 的多个目标指标（含振幅）。
- 模型：轻量回归（特征→终值），多目标。

## 3. 怎么标定（关键——决定可不可信）

### 3.1 LOSO（留一交叉验证）
在 **3 个 sC 冷启动的完整 80 代跑**上做 LOSO：每次留一个 seed 当验证、另两个训练，看预测 gen79 的误差。
- 结果：振幅 MAE ≈ **0.0006**（相对 0.004–0.009 的量级，足够分辨配置好坏）。

### 3.2 方向性校验（direction-of-effect）
不只看绝对误差，还看"预判器对某个旋钮的效应方向判断对不对"——如果真跑里加大某参数振幅升，预判器也应预测升。方向一致才敢用它筛配置。

### 3.3 标记不可靠的早期信号
有些早代信号本身噪声大、对终值预测力弱（如早期 coverage / hunt 类指标）。**这些被显式标记为"不可靠"，不纳入筛选判据**，避免被噪声带偏。

## 4. 有效边界（最重要的教训）

预判器是在 **sC 冷启动（gen0–15→gen79）** 上标定的。它**只在这个工况内可信**。

⚠️ **踩坑警告**：把它直接套到"从 gen80 **热 resume** 的长跑"是**未验证的迁移**——
- 冷启动 vs 热 resume 的早代特征分布不同；
- 预判器没在热 resume 工况标定过 → 它给的方向可能是噪声；
- 贸然据此开新短跑会白烧 CPU 还得出错误结论。

所以在 sD_warm（热 resume）长跑上，**没有贸然用早代预判器开新短跑，而是让长跑自己推进到 tail 拿真振幅**。

## 5. 跑测试验证

- LOSO MAE 0.0006（3 个 sC 完整跑）= 标定通过，可用于 sC 冷启动工况的配置筛选。
- 热 resume 工况：未标定 → 不用 → 改为等真振幅。

## 6. 一句话给接手模型

**早代预判器能从 gen0–15 预测 gen79（LOSO MAE~0.0006），让冷启动配置筛选进入分钟级。但它只在被标定过的工况（sC 冷启动）内可信——换成热 resume 必须重新标定，否则方向是噪声。永远先问："我这个预判器在当前工况上验证过吗？"**
