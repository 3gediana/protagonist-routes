# 两个世界·机制拟真审计 + 洞穴持久化设计

目的：把"广但浅"的机制逐条做到拟真。每条标注 **现状(代码位置) → 假在哪 → 拟真升级 → 重训代价**。
重训代价分两档：**[免]** 不破冻结契约(只改世界/动力学/奖励，热启续训) / **[批]** 需要新感知或新动作(攒一批一次性重训，DP≈3h一条 lineage)。

---

## A. PE 世界（NEAT 群体演化）

### A1. 体温/寒冷 — `ThermalTickCapability`
- **现状**：core_temp 积分 = 季节基准(春18/夏28/秋15/冬5°C) + 昼夜delta + 篝火暖区(+1°C/s) + 运动产热(0.1°C/s·速度) − 辐射散热(0.05/s)。健康设定点37°C。
- **假在哪**：① 体温对**所有人一视同仁**，无个体差异/无衣物隔热项；② 散热不分地形（树下/洞里/旷野一个样）；③ 风/雨/雪天气不进散热（WeatherLayer 存在但只喂闪电点火）。
- **拟真升级**：**[免]** ① 树冠/洞穴/建筑降低散热系数（遮蔽=保暖）；② 天气接入散热——风increases散热、雨雪加剧失温；③ 衣物(已有 clothing 里程碑物)作为隔热乘子。
- 这条是 PE "预判趋火节律"的根，做真了 → 节律才有被逼出来的物理基础。

### A2. 洞穴 — `TerrainLayer::inCave` / `CaveMask` / `CaveHideCapability`
- **现状**：洞穴是 TerrainLayer 里的噪声掩码(CaveMask)，有 `cave_shelter_factor` 参数；**但当前唯一用途**是 CaveHideCapability 把 small_forager 标记为"在洞中"→ 对洞外观察者隐身。**对体温/存储/掠食毫无作用**，且**从未导出到 viewer**（所以画面里根本看不见洞）。
- **假在哪**：洞穴名存实亡——只是给小型猎物开的"隐身格子"，主角住不住洞没区别。详见下方 §C 持久化设计。

### A3. 火/篝火 — `CampfireLayer`
- **现状**：火是**预置在固定点位**的火堆，初始休眠，靠**雷暴闪电**概率点燃(`igniteNextDormant`)；暖区(+能量/暖体温)、危险区(烧伤致死 Combat)。
- **假在哪**：① 火**不是主角点的**——是世界/天气给的，违背你"起火是主角技能、按需出现"的本意；② 火不灭不弱（无燃料消耗，点着就常明）；③ 数量/位置固定，不能在需要处生火。
- **拟真升级**：① **[批]** 新增"生火"动作(NEAT 加一个输出端) → 主角在有柴(stick)处按需点火；② **[免]** 火有燃料条，烧柴随时间衰减、需续柴，无柴则灭——火变成要维护的资源；③ **[免]** 雨雪降低火强/熄灭。

### A4. 狩猎 — `ProtagonistHuntCapability`
- **现状**：output[0]>0.3 触发，半径内最近目标扣血、攻击者回能；武器乘子(矛1.6/石1.3/棍1.1/徒手1.0)；大型食草有"群伤阈值"(单人单击不破阈值，要持续/多人累积伤害)；猎物死→Predation、记 kill、HRL 子目标8。
- **假在哪**：① 伤害是**瞬时距离判定**，无朝向/无突袭加成/无追逐耐力；② 猎物被打**不还击/不逃散群体响应**(个体逃，但无"惊群")；③ 武器只改伤害倍率，无耐久/无投掷-近战区别在此处。
- **拟真升级**：**[免]** ① 背后/静止突袭(已有 stealth 标记)给暴击；② 命中触发猎物群体惊散(广播威胁，类似你给 DP 加的"钓鱼招狼"反向版)；③ 大型猎物反扑伤人(危险-收益)。

### A5. 潜行/隐蔽 — `StealthCapability`
- **现状**：静止≥阈值秒数 → isStealth=true；移动即破。
- **假在哪**：只看"静不静"，**不看地形遮蔽**(草丛/林下/洞中应更易潜行)、不看光照(夜里更隐蔽)。
- **拟真升级**：**[免]** 潜行成功率 = f(静止, 植被遮蔽, 夜晚, 逆风)。与 A1 遮蔽场共用一张"遮蔽图"。

### A6. 建造 — `WorksiteLayer`（系统其实很深，被低估了）
- **现状**：4 种建筑(Wall墙/Shelter棚/Storage仓/Lookout瞭望)。工地需 stick+stone 配额，主角搬运沉积材料 → assembly 进度推进 → 完工。完工效果**真实且分化**：Storage 持续吸收材料、Lookout 把附近感知半径×1.5、完工工地变成**掠食者撞墙障碍**(`enforcePredatorObstacles`)。建筑类型由附近主角**投票**决定。
- **假在哪**：机制深，但 **viewer 全没体现**——进度字段(stored/required)没导出(w[4]恒0)，4 种建筑外观不分、无搭建过程(我已做好分阶段小屋模型，缺真进度数据喂)。
- **拟真升级**：**[免/工具]** ① 改 TraceRecorder 导出 stored_sticks/stored_stones/required/completed/type → 喂我已做好的分阶段小屋；② viewer 按 4 类型给不同模型(墙/棚/仓/塔)。

### A7. 进食/能量/补水 — `SurvivalStatusLayer` + Drink/Hunt
- **现状**：health/hydration 双槽；脱水/失温/饥饿/火烧四死因取最大severity判定死亡归因。
- **假在哪**：① 食物只是"能量数字"，无种类营养(蛋白/维C)差异(DP 有，PE 无)；② 无食物腐败/储存衰减(与 Storage 建筑该联动)。
- **拟真升级**：**[免]** 露天食物随时间腐败、Storage/洞穴内保鲜 → 把存储建筑和洞穴的价值焊死。

## C. 洞穴持久化设计（核心——让洞穴"永不过时"）

你的本意：主角先住洞 → 后出来盖房；但在 PE 那个**连续世界**里要一直有用。
做法：给洞穴 **4 个不可替代且贯穿全程**的功能（基础设施 CaveMask/shelter_factor 已存在，主要是接上动力学 + 导出渲染）：

1. **最强热缓冲**（接 A1）：洞内 ambient 向"地中恒温"(~15°C)收敛——夜里最暖、正午/盛夏最凉。冬夜露天必失温，洞内能熬。御寒/避暑双向刚需，草棚/房子都比不上洞。
2. **防掠食安全睡眠**：洞口窄、可守；洞内不被狼/鹰锁定(复用 inCave 隐身，扩展到掠食目标选择)。夜间在洞=安全恢复。
3. **不腐存储**（接 A7）：洞内存粮/火种不腐烂、不被偷；露天会坏会被抢。早期没 Storage 建筑时，洞就是天然仓。
4. **聚落锚点**（连续世界）：洞穴累积存粮+长明火种，成为部落的"家/重生/集结点"——即便盖了房，洞仍是"最安全+储藏+火种"的核心。

**落地步骤**：
- **[免] 动力学**：ThermalTickCapability 读 inCave→ambient 拉向恒温；SurvivalStatusLayer 洞内回血/不被掠食锁定；食物腐败在洞内归零。
- **[工具] 渲染**：TraceRecorder/world dump 导出 CaveMask → viewer 在山体侧面画洞口(黑色拱形+内部暗腔)，让"必须得有"的洞**看得见**。
- 不破契约(无新 obs/动作)，可进下一份 PE exe + 新 seed，不动当前3个在跑的 seed。
