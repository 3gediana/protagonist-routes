# Protagonist Ecology · 最终想实现的效果

> 这份文档 **只属于** `routes/protagonist_ecology/` 这个当前项目。
> 它不是根目录 `docs/` 的补丁，不考虑未来并回其它方向时的兼容性。
> 可以把它理解为：**当前目录下完整推进的项目主体**。

---

## 一、这条路线最终想做成什么

一句话：

**把它做成一个“神经进化驱动的原始聚落沙盒世界”。**

它不是一个只会觅食和逃跑的生态箱，也不是一个纯研究型进化实验器。
它最终应该呈现出的是：

- 有主角感的智能群体
- 有真实地形和环境风险
- 有资源、搬运、堆放、回巢和协作
- 有基地感、物流感、聚落感
- 有代际演化带来的行为进步
- 既能离线高复杂度演化，也能实时交互干预

用户最后看到的，不是一堆随机移动的点，而是：

**一群带大脑、带记忆的主角个体，在野外采集、回基地堆料、围绕火点和庇护所组织生活，并逐渐出现原始分工与聚落秩序。**

---

## 二、它在观感上应该像什么

### 2.1 第一眼的感觉

用户打开系统后，应该立刻感觉到：

- 这不是实验曲线工具
- 这不是一堆自动寻路 NPC
- 这不是普通生态模拟

而是一个：

**正在“活着”的原始沙盒世界**。

地图中应该同时存在：

- 高地 / 低地 / 洞穴 / 可燃区域 / 资源区 / 危险区
- 主角群体
- 环境背景生物
- 可搬运物体
- 基地活动中心
- 物资流动与局部聚集痕迹

### 2.2 盯着看几十秒后的感觉

用户应该能逐渐看懂这些秩序：

- 有些个体在外出搜寻资源
- 有些个体在往固定区域搬运物资
- 有些个体长期在基地附近活动
- 夜晚或危险时，群体活动范围明显收缩
- 某些区域逐渐形成库存点 / 火点 / 安全点 / 工作点
- 主角群体明显比背景物种更“像在经营一个地方”

系统应当传达出的核心观感是：

**它们不是在乱跑，而是在“组织生活”。**

## 三、最终世界里最重要的 6 个观感闭环

### 3.1 资源闭环

**采集 -> 搬运 -> 堆放 -> 再利用**

这是整个系统最重要的沙盒闭环。
如果没有这一层，它就永远只是生态模拟。

### 3.2 地点闭环

**野外 -> 路线 -> 基地 -> 库存点 -> 工作点 -> 庇护点**

“地点有功能”之后，地图才会从背景图变成世界。

### 3.3 风险闭环

**夜晚 / 捕食者 / 火 / 恶劣天气 / 疾病 / 地形阻隔 -> 逼迫群体回巢、储备、避险、重构路线**

没有风险，就没有基地必要性，也没有真正的生存感。

### 3.4 劳动闭环

**谁去找东西、谁去搬、谁留守、谁处理重物、谁在边界活动**

不必强行做脚本职业系统，但必须出现“原始分工感”。

### 3.5 地形闭环

**利用地形 -> 改造地形 -> 新的庇护或路径 -> 反过来影响行为**

地形不是装饰，而是行为结构的一部分。

### 3.6 代际闭环

**前代更乱、更脆弱 -> 后代更会回巢、更会搬运、更会利用地点和协作**

用户必须能看出“进化前后真的不一样”。

### 3.7 建造闭环（emergent shelter）

**采集材料 -> 搬到工地 -> 工地完工后真的变成物理实体 -> 挡住外界生物 -> 主角自己却能进出**

用户原话（2026-05-22）："他们把材料收集到工地过后，但他们不会去用这些材料，真的去把自己的窝改造成一个可以抵御外界生物的这么一个地方。就好歹你给它设计成就是你把那个材料给它堵在那个窝前面，然后堵着那些其他生物，不让进来，只能让他们进来。"

这条闭环是这条路线**最终防御机制的归宿**。它不应该是 hardcoded 的"狼进窝就掉血"规则，而应该是：

1. 主角从野外砍树/捡石头收集材料
2. 把材料搬到工地堆放（已有）
3. 工地完工后，材料**真正变成物理墙体**（在 nest 入口、围拢、留缝）
4. 这些墙体作为 collidable obstacle 阻止 predator 移动
5. 主角自己能从缝隙穿过（要么因为体型小，要么因为有一个 friend-or-foe 判断）
6. 不需要再加任何"魔法伤害"——狼进不来纯粹是因为路被堵了

**当前状态（2026-05-22 夜更新）**：~~`BaseLayer.nest_predator_damage_per_second` 是一条临时占位规则~~ 已删除。**L1 机制底座 + L3 物种识别已完成**：`WorksiteLayer::enforcePredatorObstacles` 让完工 worksite 变成 collidable 圆，predator 撞墙被推回边缘、protagonist 不阻挡（friend-or-foe 直接由 `Agent::isPredator()` 实现）。v9 retrain 后存活率反而比 hack 时还高（comm_danger_warning baseline 5.0/8 → 6.5/8 ▲30%）。但 v9 当前学到的是**直接反击狼**（attacks 0→352.5）而不是"先建墙再用墙"——eval 配置 `worksite.count=0` 让它没有建墙机会，训练 task 也没专门让 predator 浪涌前 protagonist 先建墙。所以画面上还看不到"墙挡狼"的高光，是训练 task 设计问题，不是机制问题。

实施分级（B 阶段路线）：
- **L1 单墙** ✅ **机制完成**：worksite 完工后形成 collidable 圆（`WorksiteLayer::enforcePredatorObstacles`），predator 撞墙被推回边缘、protagonist 不阻挡。`AgentSocialPerception` 加 peer_count / predator_count，`ProtagonistHuntCapability` 开放 predator，`shelter_camp_reward` 上线。**画面行为部分**（让 v9 真的学会"建墙后躲在里面"）需要 L2 训练 task 设计才能出来。
- **L2 围拢 + 训练 task 改造**：多个 worksite 沿 nest 边缘排列、完工形成一圈墙留 1-2 个口；让 comm_danger_warning eval 加 worksite preactivate，让训练 task 设计专门让 predator 浪涌前 protagonist 先建墙。
- **L3 物种识别** ✅ **L1 已附带完成**：`enforcePredatorObstacles` 用 `isPredator()` 区分，protagonist 不受墙影响。
- **L4 损伤与维护**：墙体可被 predator 攻击破坏，protagonist 必须周期补修
- **L5 升级**：木墙 -> 木+石墙 -> 多层墙，提供不同保护级别

L1 机制底座已经解锁这条闭环；L2 是把"行为画面"补齐的训练 task 改造；L4-L5 是视觉和故事感上的进阶。

## 四、最终想看到的几个高光画面

### 4.1 夜晚回巢 / 围火收缩

白天主角分散活动，夜晚逐渐向安全区或火点附近收缩。
这会让系统第一次具有“家”的感觉。

### 4.2 库存点形成

野外资源被一件件搬回，时间一长，地图上真的形成堆场或库存点。
不是面板数字，而是肉眼可见的物资聚集。

### 4.3 重物协作

某些物件需要多个个体共同接近才能搬动，或某些地形重活需要多个体协作完成。
这是群体协作感最直接的高光。

### 4.4 地形改造留下文明痕迹

个体逐渐挖浅坑、垫小坡、清出通道、改出避难点。
地图会留下长期活动造成的空间痕迹。

### 4.5 前后代对比明显

早期个体行为杂乱、路线不稳定、常死。
中后期个体更会搬运、回巢、利用库存点和安全区。

### 4.6 第一次有家可守

最终用户应该看到的关键画面：

主角群把砍下的树枝、捡来的石头一根根搬回工地堆好；某一刻工地"完工"，那堆材料**真的就在 nest 入口形成了一道墙**。
夜晚狼群冲过来——撞在墙上散开，从缝隙之间徒劳地张望。
主角们在墙后面缓步走动，第一次安心地睡了一觉。

这是这条路线区别于"普通生态箱"的标志性画面：**防御不是规则给的，是它们自己一点点搭起来的。**

## 五、这条路线未来必须同时支持的两种展示模式

## 5.1 回放观察模式（Replay Mode）

### 定位

用于：

- 跑高复杂度世界
- 离线长期演化
- 稳定做演示
- 做答辩展示片段与故事讲解

### 特征

- 世界状态来自提前跑好的结果
- 用户可以暂停、快进、拖动时间轴、切换视角
- **不能改变世界演化本身**，因为它是回放
- 可以叠加：热力图、事件时间线、谱系、行为统计、镜头脚本

### 优点

- 最稳
- 可承载最高复杂度
- 最适合呈现“几十代 / 上百代后的最终状态”
- 最容易做出电影感和答辩观感

### 最适合展示的内容

- 聚落形成过程
- 代际差异
- 重大事件回放
- 资源流动与路线形成
- 物种竞争与衰亡

---

## 5.2 实时干预模式（Live Intervention Mode）

### 定位

用于：

- 课堂演示互动
- 观察即时反应
- 验证系统对事件的弹性与行为改变
- 提供“上帝干预”的可玩性

### 特征

- 世界正在实时运行
- 用户可以往系统里投放事件
- 复杂度可适当低于离线回放模式
- 强调可观察反应，而不是最长时间尺度的演化

### 可投放事件示例

- 闪电 / 火种
- 新资源点
- 捕食威胁
- 疾病爆发
- 地形阻断
- 暴雨 / 寒潮 / 干旱
- 洞穴塌方 / 落石

### 优点

- 很有互动感
- 老师和观众能直接参与
- 能展示系统不是预录动画，而是真的有内在机制

### 风险

- 更吃算力
- 更容易被某次随机事件打乱观感
- 需要对系统复杂度做更谨慎的在线预算

## 六、复杂度的合理终局

这条路线的合理目标，不是无限堆 feature。
而是达到一个**高复杂但仍可观看、可解释、可训练**的状态。

### 推荐终局规模（只是方向，不是死数）

- 地图：中到大尺度连续世界
- 主角群：少量但明显更强的个体
- 背景实体：较多，但脑更小、角色更明确
- 多个生态位 / 竞争物种 / 威胁源
- 对象世界：可拾取、可掉落、可堆放、可协作搬运
- 地点世界：巢区、库存点、火点、工作点、庇护点
- 风险系统：夜晚、天气、疾病、捕食、火、地形障碍

### 合理上限判断

它可以复杂到：

**“原始聚落 / 前文明沙盒模拟”**

但不应该盲目追求：

- 无穷多物种
- 过细化学 / 流体 / 全物理
- 全体个体都用超大脑
- 超长制作树
- 纯 feature 堆砌

最强的复杂，不是数量，而是：

**少数高价值机制形成很多肉眼可见的闭环。**

---

## 七、非目标（这条路线不追求什么）

为了保证项目最终真的能收敛，这条路线明确不把下列内容设为刚性目标：

- 完整复刻 Dwarf Fortress / RimWorld / Minecraft
- 一上来就做全功能实时大世界编辑器
- 让所有物种都拥有同等级复杂脑
- 在没有聚落闭环之前先做过重 UI
- 把“指标变多”误认为“世界更真实”

---

## 八、什么算这条路线成功了

如果最终系统达到下面这些效果，我就认为这条路线是成功的：

### 8.1 用户一眼能看懂亮点

用户不需要读代码、不需要看长文档，只看屏幕就能理解：

- 这些个体在采集
- 在搬运
- 在回基地
- 在利用地形与庇护点
- 它们和普通背景生物明显不同

### 8.2 有明显的“地方感”

地图上真的形成：

- 活动中心
- 库存点
- 路线
- 避险区
- 长期痕迹

### 8.3 有明显的“群体生活感”

不是所有个体各做各的，而是能看到：

- 物流
- 弱分工
- 局部协作
- 集体收缩 / 扩张
- 资源流转

### 8.4 有明显的“代际变化感”

前后代行为差异可被肉眼观察，不只是 fitness 曲线上升。

### 8.5 两种展示模式都成立

- **回放模式**：能稳定展示高复杂状态
- **实时模式**：能实时注入事件并看到反应

## 九、当前路线的下一步机制优先级

为了逼近上面的最终效果，接下来机制优先顺序应当是：

1. **BaseZone / NestZone / Stockpile**
2. **ObjectLayer 接入 protagonist route**
3. **PickUp / Drop / Carry 闭环**
4. **回基地存放的收益闭环**
5. **Fire / Shelter / Night return**
6. **Heavy object / cooperative labor**
7. **Terrain modification 进入基地建设闭环**
8. **简单 workshop / crafting 钩子**
9. **回放与实时干预双模式分化**

---

## 十、最后一句话

这条路线最终想实现的，不是“更复杂的生态指标系统”，而是：

**一个可被观看、可被讲故事、可被回放、也可被干预的神经进化沙盒聚落世界。**

用户最后看到的，应该像是：

**文明出现之前，一群带记忆和可进化大脑的主角个体，正在学会如何在一个真实世界里共同生活。**
