# 指标画面感筛选 (Metric Depictiveness Study)

> 标准（用户定义）：**好指标 = 仅凭这一个值（或它的变化量 Δ），脑中就能自动浮现整段训练的画面，不需要再去猜。**
> 靠脑补才能成像的 = 坏指标。本文把 ~30 个候选指标逐个过三关，只保留能独立成像的。

数据来源（全部用户真机真实历史，零改引擎）：
- **PE**：`onepot_sD_warm_50024 / 20260605-094058`，merge 后 **97 代**（fitness+behaviors+evolution）
- **DP**：`ft_wb.jsonl`，**952 局**（每集 reward 分解 / unlocks / act[23] / 死因 / 夜间避寒）

## 三关（把"画面感"这个主观词变成可判定的硬条件）
1. **有方差**：全程几乎不动 → 零画面（静态值反映不出动态）。用 CV（变异系数）判，CV≈0 或 DEAD = 砍。
2. **独立**：与某个已保留指标 |r|≥0.85 → 同一幅画的两种写法，塌成一个。
3. **能写出一句具体场景**：写得出"这个值 = 什么画面"才留；写不出 = 没画面感。

## 由画面感筛出的"更强指向性指标"（复合，每个都自带画面）

这些不是又一堆标量，而是**把上面保留项压成少数几个自带画面的指向标**：

### 1. DP 专注-存活耦合（"世界融贯"核心指标）
```
focus_survival = z(survival) − z(action_entropy)      # 单局或滑窗
```
- **画面**：正且大 = 笃定地做对的几件事并活着（真的理解了世界的因果）；负 = 慌乱乱按、开局即死。
- **实证**：熵↔存活 −0.62（n=952）；候选 COHERENCE↔ms_depth **+0.61** → 与主线深度强同向。

### 2. DP Goodhart 缺口（反刷分哨兵）
```
goodhart_gap = milestone_reward_share − survival_norm
```
- **画面**：缺口大 = 在猛领里程碑/势能分，却活不下来 = **假进步**。本run r_milestone 是最大奖励项之一而 survival≈0.6% → 缺口拉满。

### 3. PE 机制胜任度（够不够得着世界）
```
competence = geomean(chop_rate, craft_rate, throw_rate, hunt_rate)
```
- **画面**：地板值 = 可达性灾难（砍不动/造不出/打不到）；回升 = 种群真的开始"会用"世界的机制。

### 4. PE 广度×胜任 协同（识别"虚假多样性"）
```
healthy_diversity = me_coverage  AND  competence 同向上升
fake_diversity    = me_coverage 高  BUT competence 地板
```
- **画面**：这正是 `panel_demo_long_50027` 的病——**niche 7/7、me_cov 健康，但 craft 0.014%、信号响应 0%**：铺开了生态位却全都够不着机制。单看 me_cov 会被骗，**两者并看才有真画面**。

## 跨切面关键洞察（这是"以前看不出方向"的根因）
1. **DP 的"动作丰富"= 快死了，PE 的"行为多样"= 健康**——方向相反。以前把"多样性"当统一好信号，正是误判来源。DP 单体要的是**收敛专注**，PE 群体要的是**铺开 + 每格都能执行**。
2. **冗余确实存在且可证**：PE 的 QD ≈ avg/best（r≈0.84），DP 的 entropy≈repert、r_death≈r_alive（均 +0.78）。20 个数里真正正交的不到一半。
3. **有些指标只在"坍缩瞬间"有画面**（niche_occ、sig_resp）：不该作连续指标天天看，应作**绊线**（越界即触发回退/告警）。
