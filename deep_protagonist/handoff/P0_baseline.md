## 两系统并排：墙是同一面

| | DP | PE |
|---|---|---|
| 机制宽度 | 8–10 / 23 动作，2–3 / 10 里程碑 | 1 / 4 核心机制真能成，coverage 0.2–0.32 |
| "空按"现象 | tendfire 416×/局却冻死 | chop/craft/hunt 各 5–6 万次，成功≈0 |
| 共同死区 | 科技树·造屋·种地·驯养 全 0 | 生火·喝水·近身狩猎·通讯 全 0 |
| 近身狩猎 | prey_kills 偶发，大部分局 0 | 0.002% |
| 通讯/协作 | （DP 单体无此项） | signal_response ≡ 0 |
| 存活 | 0% 活到底，多为冻死 | ~13% |

**一句话**：两边都不是"没去试"，而是**绝大多数机制要么机制上够不着（成功率≈0）、要么够得着但没收益压力（喝水/生火/造屋不做也暂时死不了/不影响峰值适应度）**，于是各自坍缩到一条最省力的活路（DP=eat+collect+spear+spam-fire；PE=投掷杂食）。这正是"标量爬坡 → 单策略坍缩"。
