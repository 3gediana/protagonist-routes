# 训练方案：从"单指标爬坡"转向"全机制涌现"

> 缘起：你给的方向 ——"尽可能全面抓，不要盯着一两个指标，还有那么多机制呢"。
> 这份方案的核心就是把目标从"把 kills/fitness 这一个数字推高"换成"让尽可能多的机制（采集/捕鱼/烹饪/造屋/采矿/造工具/用火/种植/驯养/建纪念碑/狩猎…）都被用起来、并各自做好"。

---

## 0. 一句话诊断：我们一直在"标量爬坡"，所以塌成单一策略

- **DP**：单智能体 PPO，奖励是一个标量。标量优化的天然结局就是**坍缩到一个局部最优策略**——它找到了"窝在角里 eat+craft 糊口"，于是再也不动。我们之前所有努力（趋近奖励、稀缺、随机出生、覆盖奖励）都是在**同一个标量上加项**，本质没变，所以反复塌回去。
- **PE**：多智能体 NEAT，适应度也是标量峰值。结局同样坍缩——进化选出了"投掷型杂食通才"，把 apex/pack/ambush 这些专精生态位全washed out（你看到的"赢家是通才不是猎手"）。

**这正好对应学术界一个成熟的判断：要"多样涌现"，就不能优化一个标量，要优化"行为空间的覆盖度"。** 这套范式叫 **Quality-Diversity (QD) / MAP-Elites**：不求单个最强，求**填满一个由不同行为定义的格子档案**，每个格子里保留一个"在那种行为上最好"的个体。

---

## 1. 统一原则（两个系统都适用）

> **度量和选择"机制覆盖度"，而不是某一个标量。**

落地成三件事：
1. **先做"全机制涌现面板"**（不训练，纯埋点）：每条命/每代，统计每个机制是否被用到、用了多少次、做成没有。让我们能在**分钟级**看到"涌现谱"宽不宽——这是你要的"全面抓"的眼睛。
2. **把"覆盖度"变成选择压力**：DP 用势能塑形（PBRS）奖励"这条命用过的不同机制的数量"；PE 把现有 niche 系统升级成真正的 MAP-Elites 档案（每个 niche 各留一个 elite，不让通才吃掉专精）。
3. **守反-Goodhart**：覆盖度是"用过几种不同机制"的单调有界势函数，不改变最优策略、只引导广度——**不是对单个机制发赏金**（那会重蹈"刷某个数"的覆辙）。

---

## 2. DP（单智能体）方案：把"狩猎塑形"推广成"全技能覆盖塑形"

### 2.1 现状可复用的资产
- `ft_bc10.pt`：BC 底座 + 微调，**已经长出击杀**（最近 40 局 5 局有杀）。这是"狩猎"这一格的种子，别丢。
- `phi_hunt_progress`（已实现的狩猎势函数）：现成的 PBRS 框架，**直接推广**。
- 23 动作 / 582 观测 / 网络 I/O 是红线 —— 下面所有改动**零维度变化、resume 安全**。

### 2.2 全机制清单（DP 的"涌现谱"，共 ~15 个机制格）
生存：① 采集果子 ② 喝水/捕鱼 ③ 吃熟肉
火与烹饪：④ 生/续火 ⑤ 烹饪
建造：⑥ 搭庇护所 ⑦ 蓝图工地 deposit ⑧ 建纪念碑(大熔炉)
工具/衣物：⑨ 造矛 ⑩ 草衣 ⑪ 皮披风 ⑫ 石斧 ⑬ 石镐
采矿：⑭ 挖矿
农牧：⑮ 种植/浇水 ⑯ 驯养
狩猎：⑰ 近身击杀猎物（HUNT_V2，已通）

### 2.3 改法（两步，都 resume 安全）
**第一步——面板（必须先做）**：在每回合诊断行里加"本回合用到的机制位图 + 各机制成功计数"。用一个聚合脚本读最近 N 回合，打印"涌现谱宽度 = 平均用到几种机制"。先 baseline 一下 `ft_bc10`：它现在到底只会几种？（大概率仍很窄）

**第二步——覆盖势函数 `phi_repertoire`**：
```
phi_repertoire(s) = (本条命已成功使用的不同机制数) / (机制总数)   # 单调、[0,1]、有界
reward_shaping += coef * (gamma * phi_sp - phi_s)                  # PBRS，不改最优策略
```
- 它只在"第一次用成一个新机制"时给一次性势能跳变 → 引诱**广度**（去试没试过的机制），而不是grind单个。
- 和现有 `phi_hunt_progress` 叠加（狩猎那一格仍有细粒度牵引）。
- 从 `ft_bc10` resume，跑随机出生 + HUNT_V2 世界，盯**面板宽度**爬升（而不是盯 kills 一个数）。

> 备选（更重，看资源）：跑 N 个 PPO 实例，每个用 descriptor-conditioned 奖励偏置到不同 niche（一个偏捕鱼、一个偏建造…），存成档案——这是 PGA-MAP-Elites 的单智能体版。先做轻量的 `phi_repertoire`，不行再上这个。

## 3. PE（多智能体生态）方案：它本来就是 QD 的天然土壤

PE 已经有 niche 系统（apex/pack/ambush/scavenger/omnivore/herbivore/forager）——**这已经是一个原始的行为档案了**，只是现在用标量适应度选择，导致通才吃掉专精。四个升级：

### 3.1 把 niche 系统升级成真正的 MAP-Elites（最对症）
- **每个 niche 各保留一个 elite**（niche 内精英主义），跨代不被通才washed out。
- 选择/繁殖时按"填满空格子 + 提升各格子质量"驱动，而不是只比全局峰值。
- 直接修复你看到的"apex/pack/ambush 全平、只有 omnivore 涨"——因为现在通才不再能挤掉专精。
- 这是 MAP-Elites / 带局部竞争的新颖性搜索的标准做法，PE 改动相对集中（动选择/存档逻辑，不动个体网络）。

### 3.2 移植 HUNT_V2 两阶段狩猎机制（拆掉那堵和 DP 一模一样的墙）
- PE 的近身狩猎死（0-2/万次），和 DP 撞同一堵墙：猎物太快、一近就逃 → 只能绕道扔投射物。
- 把 DP 刚验证成功的两阶段机制（平时正常走、被打才短冲刺、近身不恐慌）搬到 PE 的动物里 → apex/pack/ambush 这些"近身捕食"niche 才**物理上可达**，进化才可能填它们。

### 3.3 给"通讯"造选择压力（这是 signal_responses≡0 的根因解）
文献结论很硬：通讯不会无缘无故涌现，它需要三个条件同时成立——
1. **信息不对称**（部分可观测）：某个体能看到食物/危险/猎物，另一个看不到；
2. **一个非协作不可的任务**：比如**需要 ≥2 个个体才能放倒的猎物**（合作围猎），单个怎么都打不下来；
3. **接收方有遵从的动机**：回应同伴信号能真的提高自己的适应度。
- 现在 PE 有信号通道但**没有任何任务需要听信号**，所以进化正确地忽略它（emits=48 全程没人回应）。
- 加"合作围猎猎物 + 信息不对称"后，"发信号叫同伴 + 回应信号一起围"才会成为高适应度策略 → 通讯/协作有机会从收益里长出来（而不是发奖金硬逼）。

### 3.4 自动课程（autocurricula）
- PE 是多智能体 → 一旦有了 3.3 的合作/竞争压力，会像 hide-and-seek 那样产生**自驱动的军备竞赛课程**（一方进步逼另一方适应），文献证明这比"内在好奇心"更能逼出"人类相关"的复杂技能。这是 3.1–3.3 落地后自然涌现的红利，不用单独写。

## 4. 度量纪律（你们自己文档里的两条铁律，别再踩）

- **PE 指标写盘时机**：振幅/trace 只在每个长跑最后 20 代（tail_generations=20）写盘 → 短跑/前期读不到。做覆盖度面板时**必须让它每代都写 niche 占用 + 各行为计数**，否则又是"以为什么都没发生"。
- **先量正确的中间量再改**：DP 的 kills=0 当年卡死，是因为没人量"离猎物多近"。这次先把"涌现谱面板"做出来，**用'改了没动'当证据**，别再猜参数。

## 5. 建议的执行顺序（资源感知：DP 吃 GPU、PE 吃 CPU，可并行）

| 阶段 | DP（GPU） | PE（CPU） | 看什么 |
|---|---|---|---|
| P0 仪表 | 加全机制面板 + 聚合脚本 | 加每代 niche 占用 + 行为计数写盘 | 分钟级看"涌现谱宽度" |
| P1 覆盖 | `phi_repertoire` PBRS，从 ft_bc10 resume | niche→MAP-Elites 档案（每 niche 留 elite） | 面板宽度↑ / 被填的 niche 数↑ |
| P2 机制 | （继续 P1 长跑） | 移植 HUNT_V2 + 合作围猎/信息不对称 | apex/pack 可达？signal_responses 冒头？ |
| P3 长跑 | 跑满，按"覆盖度"评估 | 跑满，按"填满几个 niche"评估 | 不再看单一标量 |

**评估口径换掉**：成功 = "用到/做成的机制种类数"上去了、"被填满的 niche 数"上去了，而不是某一个 kills/fitness 数字。
