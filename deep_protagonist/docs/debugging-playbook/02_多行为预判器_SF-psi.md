# 案例 02：多行为预判器（23 动作面板 + Successor-Features ψ）

> 这篇讲的不是修一个 bug，而是**为什么"快速迭代"必须先有"分钟级读信号"的工具**，以及怎么造这种工具而不破坏冠军策略。

---

## 1. 问题现象 / 动机

- 训练一次跑满要很久；如果每改一个旋钮都跑满才看结果，迭代速度趋近于 0。
- 单看 `score` 这种聚合指标，**看不出是哪条行为链在拖后腿**（kills=0 案就是被聚合指标掩盖了好久）。
- 需要：① 分钟级看清"每个行为现在被采样多少、策略想不想做"；② 不重训就能估"换一套奖励配方会不会更好"。

## 2. 工具一：23 动作优势面板

### 做法
训练循环里每隔几千步，对当前 batch 按离散动作 id 分桶，打印每个动作的：
- `n`：采样次数（这个动作被选了多少次）；
- `adv`：平均优势 advantage（策略对它的"想做程度"，正=想做、负=被压制）。

### 价值（怎么用来定位）
- 某动作 `n=0` → 它被 gate 锁死、从没触发（如 COOK/MINE/工具在科技树没解锁时）。
- 某动作 n 很高但 adv 为负 → 策略在被迫采样它但不喜欢（早期 HUNT）。
- 某动作 n 很高且 adv 转正 → 这条行为开始被学会。

**kills=0 案的第一刀就是它**：一眼看到 HUNT 采样占比 47.6%，当场推翻"探索不足"假设，把排查引向"打猎没转化成击杀"。

## 3. 工具二：Successor-Features ψ 预判器（commit c9145e3）

### 核心思想
- 奖励是 12 个未加权通道 φ 的线性组合：`reward.total() = dot(φ, w)`，其中 w 是奖励配方权重。
- 训练一个 ψ 头预测**每个 φ 通道的折扣未来累计**（successor features）。
- 于是 `dot(ψ(s), w')` 就能**离线估**"如果换成另一套配方 w' 后这个状态的价值"——**不用重训**就能筛 N 套配方，挑最有希望的再真训。

### 关键安全性（不破坏冠军策略）—— 这是设计的灵魂
- ψ 走 **detached trunk** 的 TD 训练：梯度**只**更新 ψ 头，**绝不**回传到策略/价值/编码器主干。
- 自测验证"trunk 梯度 == none"（字节级稳定）：加了 ψ 头之后，原策略输出**逐字节不变**。
- 旧 checkpoint 容错加载：缺 ψ 头 → fresh init，靠 `sf_warmup` 收敛。**网络 I/O 一字未改，纯 append-only**（F 从 12 扩到 14：把 `vital`/`collect` 两个混合桶拆成独立线性通道，让 `dot(φ,w)=reward.total()` 严格成立且能干净 re-weight）。

### 为什么要拆"混合桶"
`reward.total()` 里 `vital` 通道实际是 decay+danger+action_cost 三样之和，`collect` 是 collected+deposited 之和。混在一起时：① `dot(φ,w)` 不严格等于 total；② 没法单独 re-weight 其中一项（比如只想加大"打猎权重"）。拆成独立线性通道后两个问题都解决。

## 4. 跑测试验证（8 项自测全绿）

ψ 不靠"碰巧训出来"来验证，而是用确定性自测**因果闭环**地证明它对：
1. **φ-recon**：`dot(φ,w)` 精确等于 `reward.total()`。
2. **ψ 不动点**：满足 Bellman/TD 不动点。
3. **done 截断**：回合结束时未来累计正确截断。
4. **re-weight 排序**：换配方后状态价值排序符合预期。
5. **零通道隔离**：把某通道权重置 0，该通道贡献干净归零。
6. **shape**：维度正确。
7. **detach 梯度**：trunk 梯度为 none（冠军策略不被扰动）。

运行：`deep_protagonist_train --sf-selftest`（或对应自测开关）。

> 诚实提醒（写进当时的交接）：另有 4 个自测 FAIL，但**不是 SF 引入的**——它们在 river/animal/wardrobe/observation 等**未被 SF 触碰**的子系统里，源于工作区里那批未提交的 obs-surgery / D-122 科技树 WIP。SF 只 commit 了自己的 8 个文件，没碰那 84 个未提交改动。**定位 FAIL 归属、不背别人的锅，也是方法论的一部分。**

## 5. 当前状态 / 注意

- ψ "机器造好了但还没通电"：ψ 头权重是随机初始化的，要给出有意义的配方预判，得在**一次真实训练里顺带把它训出来**（detached，零策略代价，搭便车）。
- 因此离线筛配方的 `--sf-eval` 工具**依赖训练过的 ψ**。接手模型若要用 ψ 筛配方：先正常训一段（ψ 会随训搭车收敛），再用 `dot(ψ, w')` 对比候选配方。

## 6. 一句话总结给接手模型

**快速迭代的前提是"分钟级读信号"。两件武器：① 23 动作优势面板（看每个行为被采样多少、想不想做）；② SF ψ 预判器（不重训估换配方的价值，detached trunk 保证不动冠军策略，`dot(φ,w)=reward.total()` 是它成立的数学地基）。造这类工具时第一原则是"绝不扰动已有策略"——用 detach + 字节级稳定自测来保证。**
