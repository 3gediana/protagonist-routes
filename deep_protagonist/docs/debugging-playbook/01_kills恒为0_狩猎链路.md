# 案例 01：kills 恒为 0 —— 狩猎链路从头到尾的断点排查

> 这是本项目最有代表性的一个案例：一个被多代人误诊的指标卡死。
> 它示范了"症状≠根因"、"先量再改"、"改了没动也是证据"这套方法论的完整应用。

---

## 1. 问题现象（精确陈述）

- 训练 228+ 回合，奖励通道 `kills` **恒等于 0**；总分 `score` 卡在 ~0.2245 不动。
- 历任改法：调奖励权重、调熵系数、调课程、调猎物稀缺度——**全部无效**，火怎么都点不着。
- 关键背景：地基锁死——**网络 I/O 不动、动作空间 23 不变、观测 582 维不变**。

## 2. 误诊史（为什么之前都没解决）

所有人**默认** kills=0 是"探索不足"或"奖励配方不对"，于是都在调 RL 超参。**没有人去量"智能体到底离猎物多近、有没有命中、命中后发生了什么"**。错误的假设 + 缺失的中间量 = 永远绕不到根因。

## 3. 确定原因的方法

### 3.1 第一刀：23 动作优势面板（推翻"探索不足"）
在训练日志里每隔几千步打印每个离散动作的采样次数 n 和平均优势 advantage。结果：
- `attack`(idx2) 采样 7137 次、`craft_spear`(idx5) 15.5 万次，HUNT 动作采样占比一度冲到 **47.6%**。
- **结论：智能体根本不是不探索，它在疯狂打猎。** "探索不足"假设当场被推翻。

### 3.2 第二刀：读战斗机制代码（建立物理模型）
- 猎物血量 100，长矛每击 50 伤害 → **必须连续命中 2 下才能杀死**；第一下命中后猎物立刻逃跑。
- 兔子逃跑 6.0 m/s、鹿 7.0 m/s，智能体最高 3.0 m/s（无冲刺，沼泽更慢）。
- 猎物在 25–35m 外就开始逃。
- 初步推断：陆地狩猎"在物理上不可达"。**（注意：这个推断后来被证明只对了一半——见 3.4）**

### 3.3 第三刀：用户的认知提问，把视角从"物理"拉到"认知"
用户问了一句关键的话：**"原始人能理解猎物跑哪里去了吗？他有意愿去追吗？"**

这把排查从"追不追得上（物理）"重定向到"看不看得懂、咬不咬得住（认知）"。顺着读 `Observation.cpp` 的「4 nearest animals」段，发现两个认知缺口：
1. **只给位置、不给速度/朝向**：观测里每只动物只有相对坐标 (dx,dy,dz)+种类 one-hot，**没有速度**。智能体知道猎物此刻在哪，但不知道它往哪跑、跑多快 → 无法预判拦截。
2. **没有目标锁定记忆**：那 4 个槽位永远装"当前最近的 4 只"。被追的猎物一旦冲刺拉开、别的动物变得更近，**被追的那只就被挤出槽位 → 智能体直接看不见它了**，趋近奖励的参照点也跟着切走 → 追击意愿被打断。

### 3.4 第四刀：每步埋点，量出真实的"接近地板"（决定性一刀）
加了一个看似简单但**决定性**的埋点：`closest_prey` = 智能体每一步离最近鹿/兔的真实最近距离。

> ⚠️ 踩坑：最初我只在"按攻击键的瞬间"量这个距离——那是**有偏的**（智能体只在自认为能打到时才攻击）。改成**每一步都量**后，真相才浮现。这就是"先怀疑测量"原则的现场。

数据（131 回合，后又验证到 228 回合）：
- 智能体离鹿/兔的**真实最近距离 = 9m（史上最好的一次），中位数 21.5m，从来没进过 5m**，更别说 2.5m 攻击距离。
- 那 47% 的"命中"全打在顺手够到的鱼/狼身上；**能产肉给里程碑的鹿和兔，一次都没贴近过**。

**根因第一层钉死：瓶颈在"接近"，不在补刀、不在瞄准、不在速度。之前所有调奖励/速度的旋钮都作用在"贴到 2.5m 之后"，而智能体连那一步都到不了。**

### 3.5 为什么接近不了——读代码实锤两条
1. **观测槽位被挤占**：世界里有 30 条鱼（5–9 成群）+ 8 匹狼 + 50 兔，这些会把最近的鹿挤出那 4 个共用槽位 → 智能体看不到鹿在哪个方向。
2. **趋近奖励连带归零**：`phi_hunt_progress()` 这个"趋近猎物"的势能塑形**也是读这 4 个槽位**算的——鹿被挤出去 → 它算出"视野里没猎物" → **趋近塑形梯度=0**。于是死循环：看不见猎物 → 没拉力 → 不接近 → 猎物一直不在槽位里。
3. **代码里早有先例**：狼当年就因为"被一堆兔子挤出槽位、AI 看不见天敌"而专门加过一个**独立狼槽位（D-048）**。猎物这边一直缺这个对应物。

## 4. 调试迭代（含走过的弯路——弯路本身是教训）

| 版本 | 改动 | 假设 | 结果 | 学到 |

**三个正交方向（感知 v6 / 动机 v7 / 机制 v3-4）都撬不动同一个 9m 地板** → 这不是学习问题，是一个**内建的机械平衡点**。

## 5. 真正的根因（v8 埋点钉死）+ 最终修法（D-126）

### 5.1 根因
v8 的 `closest_rest` 埋点显示：**48 个回合里，智能体从未遇到过一只"正在力竭休息"的猎物**（`closest_rest` 始终是哨兵值）。

读耐力狩猎代码发现致命缺陷：旧逻辑里 `flee_rest`（力竭休息）**只有在智能体把猎物连续保持在 `FLEE_TRIGGER`(12m) 内满 1.2s** 才会触发；可猎物冲刺 6–7 m/s、智能体只有 3 m/s，猎物在 1.2s 没到之前就冲出了 12m 触发圈 → 进入 else 分支 → **`flee_sprint` 被瞬间重置回满**。于是猎物**永远攒不够 1.2s、永远不会力竭、休息窗口永远不打开** → 9m 地板 → kills=0。

**耐力狩猎机制此前等于是个死代码**：触发半径(12m) 小于猎物一次冲刺就能拉开的距离，所以这个机制从来没真正起效过。

### 5.2 修法：累积疲劳（D-126，守地基）
新增 `AnimalInstance::flee_stamina`（一个 float 字段，不动观测/动作/网络）：
- 逃跑冲刺时 `flee_stamina` **持续消耗**（-dt）；归零 → 强制进入休息（`flee_rest`）。
- 脱离触发圈后，stamina **只缓慢恢复**（+0.5/s），**不再瞬间重置**。
- 关键效果：**短暂逃脱不再抹掉持续追逐积累的疲劳**。只要智能体坚持追（哪怕猎物时不时冲出 12m），疲劳净累积 → 猎物最终崩溃力竭、停下 → 慢速追踪者贴到 2.5m 补刀。
- 健康猎物冲刺速度**仍 > 人**（满足用户红线"动物必须比人快"）；只有"力竭停下(vel=0)"和"受伤后(0.4×panic)"才变得可捕。这正是真实的"耐力狩猎"。

参数：兔 STAM_MAX=3.0s / 鹿 3.5s，RECOVER=0.5/s，REST_DUR 2.0/2.2s（受伤×1.8）。

## 6. 跑测试验证（v9→v11 真实进展）

判据事先写死：① `closest_rest` 出现真实读数 = 休息窗口打开；② `closest_prey` 的 min 从 9m 往 2.5m 掉 = 接近瓶颈被打通；③ `hunt_hits`>0 = 长矛开始命中；④ `kills`>0 = 整条链闭合。

逐版真实数据（`analyze_*.ps1` 聚合）：

| 版本 | 关键改动 | closest_prey MIN | rest 窗口 | hunt_hits | kills |
|---|---|---|---|---|---|
| v8 | slot-0 + shaping0.5（旧 flee 逻辑） | 9m（169 回合从未破） | **169 回合从未开过** | 偶发 | 0 |
| v9 | **D-126 累积疲劳** | 9m | **第一次打开**（2/81 回合） | 偶发 | 0 |
| v10 | **D-126b 更早力竭+喘更久**（STAM 3→2s, REST 2→3s） | **2.8m（9m 地板击穿！）** | 23/322 回合，最近 2.9m | 47 回合有命中 | 0 |
| v11 | **D-126c spear reach 2.5→3.0m** | 2.8–3.5m | 16/254 回合，最近 3.6m | 46 回合有命中、单回合最多 6 次 | 0（仍在收敛） |

**结论**：三个根因逐个被打掉——
1. 休息窗口从"永不打开"→ 频繁打开（D-126 累积疲劳，证明旧机制因触发半径过小是死代码）；
2. 9m 接近地板被击穿到 ~2.8m（D-126b 让猎物更早力竭、停得更久，慢速追踪者得以贴近）；
3. 长矛开始稳定命中猎物（D-126c reach 3.0m + 现在贴得够近）。

**当前状态（诚实）**：狩猎链路从"物理/认知上不可达"变成了"机械上可达且长矛在命中"，但 `kills` 还没稳定冒头——卡在最后一口气（接近 ~2.8m vs reach 3.0m 是临界的；命中在发生但还没凑齐"2 连击致死"）。**这已经从"环境 bug"转变为"学习问题"**：resume 的冠军 checkpoint(`r1_cook_s24`) 是在旧的"狩猎徒劳"年代训出来的，它的策略"认定"打猎没用，需要**真正的训练时长**去重新发现现在已经可行的狩猎，并学会在休息窗口里精准补刀。

## 7. 下一步（已进入"优先训练"阶段）

诊断阶段已完成使命（把不可达变可达）。接下来：
1. **锁定 v11 配置，跑真正的长训练**（不再 5 分钟一杀的诊断短跑），让策略有时间学会利用休息窗口补刀。频繁 checkpoint 存到 Windows（持久）。
2. 监控 `kills` 是否随训练时长冒头（在更长 horizon 上，策略会逐步 unlearn"狩猎徒劳"的旧先验）。
3. 若训练相当长后 kills 仍顽固=0 → 按"改了没动=排除假设"再加一档边际（reach 微调 / 提高鹿兔遭遇率，在智能体附近多生一群 / REST_DUR 再加长），但**先给训练足够时间**再动手，避免过度调参。
4. 一旦 kills>0 稳定 → commit（前缀 `deep_protagonist: fix-hunt-...`）、锁参、跑满。

> 进度文件：`runs/hunt_s24k.*`（v11），分析脚本 `analyze_v11.ps1`。逐版本二进制每次重编 `build_d122_v2/bin/deep_protagonist_train.exe`。

## 8. 一句话总结给接手模型

**kills=0 不是 RL 问题，是"智能体看不见猎物方向（观测槽位被挤）+ 趋近奖励连带归零 + 耐力机制因触发半径过小而从未真正生效"三件事叠加。修法全在地基红线内：第 0 槽固定给猎物(D-124) + 趋近 shaping + 累积疲劳(D-126)。教训：先量中间量（每步 closest_prey/closest_rest），别调超参。**

## 9. 并发假设 sweep（v-series 多变体并行，打穿"接近"墙）

用户要求"榨干算力、并发多种猜想"。把狩猎旋钮做成 **env 变量可覆盖**（`AnimalSystem.cpp::env_f` + `Environment.cpp::DP_SPEAR_REACH`，见 `04_资源上限与并发配置.md`），一个 exe 同时跑 5 套配置，独立 log/checkpoint：

| 变体 | 假设 | 配置 | closest_prey MIN | ≤3m 回合 | 单回合最多命中 | kills |
|---|---|---|---|---|---|---|
| va_ctrl | 只是训练不够 | 默认 | 2.8m | 1 | 6 | 0 |
| vb_reach35 | 差距离 | reach 3.5m | 1.8m | 2 | 6 | 0 |
| vc_fatigue | 休息窗口不够 | STAM↓ REST↑ | **0.6m** | 4 | 6 | 0 |
| vd_shape10 | 动机梯度不够 | shaping 1.0 | 2.8m | 1 | 6 | 0 |
| ve_combo | 全开（上界探针） | reach3.5+力竭++ +shaping1.0 | **0m** | 7 | 6 | 0 |

**收获**：vc_fatigue / ve_combo 把"接近"地板从 9m → **0–0.6m**（agent 真能贴到猎物身上）。**累积疲劳 + reach 是对的方向，这关算打穿了。**

**但 kills 仍=0，读原始回合数据（`raweps.ps1` 拉 closest_prey + hunt_hits + 奖励分解同行）逼出下一道墙——两条：**

1. **矛扎错目标**：prey 贴到 3.6m 的回合只命中 0–1 次；命中 2–3 次的回合 prey 都在 12–100m 外 → 那些"命中"扎的是**身边更近的鱼（成群 5-9 只）/狼**，不是兔鹿。`agent_attack` 选攻击锥内最近目标，鱼群把矛吃了。
2. **没补第二刀就死了**：回合奖励分解里大量 `bites=-13.3`（被狼咬）+ `death=-10.0`，回合频繁在 2800–6400 步就结束（满 9000）。受伤猎物（扣到 50 血）还没等到 agent 补第二刀（再 50 = 致死），agent 先被狼咬死/饿死。

**这印证了第 8 节的预测：下一道墙就是"补刀(2nd hit)"。** 现在它具体化成两个可验证子问题：
- **(下一个假设 F)** `hunt_hits` 要**按种类拆分**（鱼/狼/兔/鹿分别计数），确认贴到 prey 时矛到底扎没扎到 prey；若确实被鱼群吃矛 → 攻击锥优先选 prey，或贴近 prey 时把鱼挤出攻击判定。
- **(下一个假设 G)** 受伤猎物在被补刀前的存活窗口 vs agent 的存活窗口。若 agent 总先死 → 要么训练时长让它学会避狼，要么临时降低狩猎场景的狼密度（环境压力杠杆）让"补刀"这条链先学出来，再逐步加回狼。

> ⚠️ 注意：拆分 `hunt_hits` 需要重编译，会**杀掉所有正在跑的并发变体**。所以在"优先训练"阶段先不动，等需要新一轮诊断时统一重编、统一重启 sweep。

**当前状态（诚实，2026-06-04）**：接近已解决（0m 可达），kills 仍=0，瓶颈从"接近"下移到"补刀（扎对目标 + 活到第二刀）"。5 变体持续训练中，观察策略能否自行学会咬住同一只受伤猎物补刀并躲狼。

## D-130：停止刷指标，先看"一整局他到底在干嘛"（全生命行为轨迹）

**用户两次关键纠偏（最值钱的方法论）：**
1. *"你要训练出来的是他的能力，而不是仅仅是那个指标……他打猎一直是零，你得判断他是①不会打猎，还是②没机会打猎。"*
2. *"别先盯一个指标，先收集他一整局的行为去分析他到底在干嘛；用现有参数让他跑十分钟，看这套参数能学出什么、学不出什么；再插探针看每个行为的触发契机。机制堆了很多，先看大局。"*

**做法（不加任何奖励、不改机制）**：加 `DP_TRACE` 环境变量 → 单 env（n-envs 1）resume 冠军 `r1_cook_s24`、默认参数，**每步**落一行轨迹（位置/朝向/动作/饥渴体力/最近兔鹿·鱼·狼距离/是否力竭/有无矛）。跑满 20 万步（37 局，均长 5405 步）后整局回放分析。

**整局行为画像（200k 步硬数据）：**

| 维度 | 读数 | 含义 |
|---|---|---|
| 动作预算 | craft_spear **51.9%** + eat **33.2%** + clothes 10.6% | 离散动作头退化成"狂按造矛+吃"，维持型按钮 |

**结论——既不是单纯①也不是单纯②，而是更早的一层：他根本没走到猎物那儿去。**
- 不是"不会下手"(①)：他能贴到狼 0.01m 并出矛 → **贴身攻击能力存在**。
- 不是机制"不给机会"(②) 的字面意思：机制本可以把力竭猎物送到刀下——**前提是他得在猎物附近**，而他从不靠近。
- 真正缺失的能力 = **朝远处猎物的"长途奔袭/搜寻"**：猎物成群散在 1024 地图各处且会逃，**中位距离 85.8m > 视野 60m → 大多数时候他压根看不见猎物 → 没有任何梯度把他拉出生角**；与此同时狼主动找上门，于是他只学会了"处理送上门的东西(狼/身边食物)+ 就地糊口"，从没学会"出门打猎"。

**这一举解释了之前所有奖励为何失败**：prey-blow / fire 奖励的触发前提（猎物在 reach/视野内）几乎从不发生；灌密度之所以一度"见效"，只是把猎物**生到了他的角里**而已——治标不治本。

**下一步方向（待与用户确认，候选）：**
- (a) **遭遇率/可见性**才是真瓶颈：要么提高猎物可见半径/在 agent 视野边缘补猎物信号，要么改 spawn 让猎物分布更均匀（环境杠杆）。
- (b) **探索驱动缺失**：策略没有离开出生角的动机；需要探索 shaping 或好奇心，但这又回到"加奖励"——须先确认这是能力缺口而非机会缺口。
- (c) 先继续插探针看"他为什么不离开出生角"（出生角是否水/食物富集→留下更安全？视野外无信号→无方向？），再定方案。

> 方法论沉淀：**当一个底层指标(kills)长期为 0、且参数 sweep 全部失败时，停下来做一次"全生命逐步轨迹"回放**——它一次性暴露了 8 轮 sweep 都没看见的真相（他被困在出生角、一辈子没接近过猎物），比任何单指标都高信息量。先看大局，再下探针。

## D-131：有界「工具性」地图覆盖奖励（引诱出门，不是为好奇而好奇）

### 问题现象
D-130 整局轨迹证明根因是**他被困在出生角、一辈子没接近过猎物**（活动范围仅 420×414m / 中位猎物距 85.8m > 视野 60m）。用户明确点出两件事：
1. **砍狼是"遗传"**：狼是少数会**主动靠近** agent 的生物（轨迹实锤狼能贴到 0.01m、中位 207.7m 却时常逼近），加上早期 BC 参照脚本 `ScriptedSettler`（D-100）Layer 2「持矛+健康→打狼」的烙印 → 他把攻击几乎全喂给狼。**这不是这轮训出来的，是血统**。正确目标行为（待出门问题解决后再单独处理）：面对狼要有**决策感**——打得过且非必要就不打，除非需要狼皮做衣服。
2. **拓展活动范围 = 第一优先**，且我又掉进了"损失陷阱"：他在小角里 eat+craft 就能活得很好 → **没有任何压力出门** → 永远碰不到全图机制。需要一个**进权重的机制**引诱他走完世界。

### 用户给的分寸（关键设计约束）
> "一个机制，一个惩罚吧，该成为是权重的一部分。你只是引诱他出去，但最后训练出来的效果，他只是借用了探索这个行为……不是为了好奇而好奇，我们的好奇是为了让他熟悉世界、走完这个世界。"

即：覆盖奖励是**手段不是目的**，必须能**自我衰减**、不可被刷，走完即归零退回真实生存/狩猎，**不能喧宾夺主变成纯流浪**。

### 机制设计（守地基 582 维 / 23 动作不变，全程 env 开关）
- `RewardSystem::event_explore(v)` → 路由进 **milestone 桶**（和 `event_prey_blow` 同类，不伪装成击杀/分数）。
- `Environment::apply_action_effects_`：把 agent 位置量化成 `cell=24m` 网格，**每回合第一次**踏入某格才发一份 `DP_EXPLORE_REW`；`visited_cells_`(unordered_set) 每回合 `reset()` 清空。
  - **不可刷**：旧格子永远 0 收益 → 原地兜圈/反复横跳刷不到分。
  - **自我衰减**：走遍后无新格 → 奖励自动归零 → 退回生存/狩猎。
- 新增每回合埋点 `explore_cells`（这一局走过多少新格）+ 沿用 `closest_prey`，量"范围有没有铺开、铺开后是否开始遇到猎物"。
- **env-gated**：`DP_EXPLORE_REW` 默认 0、`DP_EXPLORE_CELL` 默认 24m → 不设则游戏奖励景观**逐字节不变**。

> ⚠️ 关键认知（别再犯）：**奖励只在「训练」中通过梯度改变行为**。给一个 resume 的**冻结策略**加奖励、然后用纯回放(eval/trace)去看，它的动作**一模一样**——因为奖励不进动作选择、只进训练梯度。所以验证覆盖奖励必须**开训练 sweep**，看 PPO 是否逐步学到"出门有收益"，而不是跑一局 trace 就下结论。

### 跑测试（剂量梯度 sweep）
都 resume 冠军 `r1_cook_s24`、参数完全一致，**仅** `DP_EXPLORE_REW` 不同：`ja_ctrl=0`(对照) / `jb=0.1` / `jc=0.3` / `jd=0.6` / `je=1.0`。判据：
1. reward 变体的 `explore_cells`（**按回合步数归一化成 cells/1k**，去掉"回合越短格越少"的混淆）应随训练**爬升**，对照 ja 应持平；
2. `closest_prey` 中位/分布应**下移**（开始遇到猎物）；
3. 生存动作预算别崩（不退化成纯流浪）。

### 当前状态（诚实记录：短horizon 内**未定论**）
跑到 ~150 回合 / 每变体 ≈0.7–0.9M 步（**仅占 8M-step 计划的 ~10%**）：
- `cells/1k` **基本持平**（2.9–3.5），对照与 reward 变体**未分离**，first8 ≈ last8 → 覆盖行为尚未被 PPO 学到。
- `closest_prey` 个别低剂量变体（jb=0.1、jd=0.6）单回合最近**蹭到 4.5–4.8m**（基线从未 <7m），但这是**单局最小值、噪声大**，非分布性改善。
- **结论：D-131 已正确编译/运行/落埋点，但其行为效果在 ~1M 步内未显现**。8M-step 是为整夜长跑设计的，10% 进度不足以让"离开出生角"的边际梯度盖过"待着也能活"的强先验。

### 如何接着验证（交接给后续）
1. **跑到接近完成**（整夜，≥4–6M 步），再用 `peek_cov.ps1` 读 **cells/1k 的时间曲线** + control vs reward 的分离度。
2. 若到中后期 reward 变体 cells/1k 仍不升 → 奖励**太小**（被"角里已能拿到 ~21 格/局"的基线淹没），把 `DP_EXPLORE_REW` 往上调一档，或把 `DP_EXPLORE_CELL` 调小（更多格、更密的边际激励）。
3. 若 cells/1k 升了但 `closest_prey` 不降 → 范围铺开方向不对（朝空地走而非朝猎物群），需配合"猎物可见性/遭遇率"杠杆。
4. **守住反-Goodhart**：一旦 `explore_cells` 飙升但生存动作预算崩了（eat/drink 占比暴跌、回合变短），说明退化成纯流浪 → 调小权重。覆盖奖励是引子，不是目标。
