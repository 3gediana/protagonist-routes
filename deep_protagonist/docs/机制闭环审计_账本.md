# 机制闭环审计 · "原始人愿不愿意干"账本 (D-122)

> 起因: cook 跑满 48M 仍=0。不是只补 cook 一个奖励就完事——要站在原始人视角,
> 系统看整条科技树是不是"做了不划算/根本不用做"。结论: **链是闭的(能做),但经济是开的(没理由做)**。

## 0. 基准线 = 一张吊床 (这是病根)
| 活法 | 成本 | 每分钟收益 | 风险 |
|---|---|---|---|
| 白天摘果子 | 走几步 | 吃一次≈+1.2 奖励,饱腹+水,可重复 | 0 |
| 盖一间窝棚后**坐在里面** | 一次性盖好 | r_alive +6/min;饥渴**净 +3/min(回血15 - 衰减12)** | 0 |

- 衰减: 饿/渴各 12/min。窝棚内回血 15/min → **坐窝棚里饿/渴反而往上涨,永远饿不死**。
- 吃的奖励 = 6 ×((100-饱腹)/100)² 的减少量,**跟食物来源无关**: 一颗果子和一头鹿,同样饱腹给同样分。
- 夜里不在遮蔽 = 3× 掉血会死 → 这是**唯一**真压力,而它被"盖窝棚"完美解掉了。
- ⇒ PPO 收敛到 "摘果子 + 盖窝棚 + 宅着" 是**完全理性**的。科技树每一级都是往这条舒服线上**额外加负担**。

## 2. 两类病灶 (你点的那两面)
- **A 类·不划算所以没人做**: 整条树都是"额外负担换一次性+1",而基准线躺着就活得好。**根因是基准线太舒服**(果子管够 + 窝棚回血),不是某个奖励调小了。
- **B 类·闭环冗余**: 打猎当场就吃饱(eat_drive 40),于是烤肉那条"以后吃"的路**天生没需求**——这是 cook=0 的直接死因,不是探索不够。

## 3. 惩罚溯源现状 (你问的"精确到哪一步")
- **死亡 -10**: 靠 PPO 的 GAE(γ,λ) 往回传,离死越近背锅越重、越远指数级越淡 → **会回溯但是一团雾,不点名具体哪步**。
- **死因已分 4 类**(饿/冻/缺蛋白/缺维生素),但**只用于日志,扣分仍一刀切 -10**,没按死因把锅摊到导致它的决策串。← **缺口**
- **已有连续溯源**: 每步"体感压力惩罚" = Σ(各指标离满的差) × 系数,**哪个指标越低每步扣越多**。这其实是比"死后扣一次"精确得多的链——锅持续落在"放任指标下滑"的那几百步。但它**把所有指标揉成一个数**,没按指标/死因拆开。

## 4. 修复方向 (全部: 只加不删 / cook_enabled 闸门内 / 守训练铁律 / 可回滚)
> 不碰 v7 世界(闸门外字节一致),不重置网络,不删旧奖励,不 nerf 现有打猎/生存(铁律)。

**修 B(让烤肉有需求,不靠直奖)** — 造一个"淡季/寒夜窗口":到点野果歇逃(夜里本就不可摘,延长/加深它)+ 体温掉得凶,这段时间**唯一**的食物/热源是"提前囤的熟肉 + 点着的火"。于是"打猎→烤→囤"变成**过冬的活路**而不是杂活;我已加的"手里有口粮"势能正好给出指向"备好粮"的梯度。

**修 A(给基准线加真压力)** — 让"宅窝棚摘果子"不再是无脑最优: 淡季窗口里果子产出降、回血不足以扛 → 必须靠储备。压力是**周期性窗口**,不是一直惩罚(不破坏老行为,加在闸门内)。

**修溯源(你的点)** — 把那个"揉成一团"的体感压力**按指标拆开**,并对每个指标加一条**提前预警的连续势能**(快没水→"水危险度"平滑抬升),让锅精确落在"导致那个具体死因"的决策串上,而非死后一刀切。PBRS 实现,有数学保证不带歪策略。
