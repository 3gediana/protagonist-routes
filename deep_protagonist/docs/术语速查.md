# 术语速查

> 大白话版。这份给用户翻。新术语必须先加这里再用。

## A · 基础概念

| 术语 | 大白话 |
|---|---|
| **神经网络** | 一个有几百万个旋钮的盒子。输入信息 → 输出决策。旋钮拧得好它就聪明 |
| **参数 / 权重** | 神经网络里的"旋钮数量"。60M 参数 = 6 千万个旋钮 |
| **训练 / training** | 慢慢拧旋钮的过程，让它做对事 |
| **推理 / inference** | 训练完后用它干活（已经不拧旋钮了） |
| **梯度 / gradient** | "每个旋钮该往哪边拧"的提示 |
| **梯度下降** | 按梯度的提示，一小步一小步拧旋钮，越拧越对 |
| **反向传播 / backprop** | 算梯度的具体方法。从输出错的地方往回追，看每个旋钮各拧错多少 |

## B · 训练相关

| 术语 | 大白话 |
|---|---|
| **batch / 批次** | 一次喂给神经网络多少样本（比如 32 帧画面一起算）。大点 GPU 算得快但占显存 |
| **batch size** | batch 的大小，常见 32 / 512 / 2048 |
| **epoch** | "把所有数据看一遍"叫一个 epoch（监督学习用）。RL 里很少用这词 |
| **step** | 训练里走一步。RL 里 = agent 在世界里动一下 |
| **learning rate / 学习率** | "每次拧旋钮拧多狠"。太大跑飞，太小慢死 |
| **optimizer / 优化器** | 决定怎么用梯度去拧旋钮的算法。Adam 是常用的 |
| **Adam** | 一种常用的优化器。"自适应"地决定每个旋钮拧多狠 |
| **checkpoint / 存档** | 把神经网络当前所有旋钮位置存到硬盘，能续训 |
| **resume / 续训** | 从硬盘读存档，从断点继续训练 |
| **loss / 损失** | "现在错得多狠"的数字。训练目标 = 让 loss 越来越小 |

## C · 强化学习（RL）

| 术语 | 大白话 |
|---|---|
| **RL / 强化学习** | "agent 自己在世界里试错"的学习方式。撞了知道是墙，得分了知道这样做对 |
| **agent / 智能体** | 那个被训练的"AI 角色"，在世界里行动 |
| **environment / 环境** | 世界本身。agent 在里面动，世界给反馈 |
| **state / 状态** | 当前世界的样子（agent 能看到的部分） |
| **action / 动作** | agent 这一步选择做什么（往前走、转头等） |
| **reward / 奖励** | 世界给 agent 的"打分"。吃到食物 +10，撞墙 -1 |
| **policy / 策略** | agent 的"打法"。给它一个状态，它输出一个动作 |
| **episode / 一局** | agent 从开始到死/到结束的一段经历。像游戏一局 |
| **PPO** | 一种主流 RL 算法。让 agent 慢慢试错，每次只允许小步改进打法，避免一次跨太大学坏 |
| **GAE** | PPO 里的辅助工具。算"这一步动作到底好不好"，把未来奖励折算回来 |
| **on-policy** | 每次只用"最新打法"产生的经验来学。PPO 是这种 |
| **off-policy** | 可以用"历史所有打法"的经验来学。SAC、DQN 是这种 |
| **replay buffer / 经验回放** | 存历史经验的仓库。off-policy 算法需要 |

## D · 神经网络结构

| 术语 | 大白话 |
|---|---|
| **MLP** | 最简单的神经网络。一层层全连通，没记忆。适合简单任务。S2 用它做 agent 的"脑" |
| **CNN** | 处理图像的神经网络，能识别"局部模式" |
| **RNN** | 有记忆的神经网络。这一刻看到的会影响下一刻的判断 |
| **LSTM / GRU** | RNN 的改进版，记忆更稳。GRU 比 LSTM 简单 |
| **Transformer** | 当前最强网络结构。能"同时关注一段历史的每个片段"。GPT 用的就是这个 |
| **attention / 注意力** | Transformer 的核心。"我现在做决定时主要看历史的哪几帧" |
| **encoder / 编码器** | 把原始输入（画面、感官数据）压缩成神经网络能用的"特征向量" |
| **embedding** | 把离散东西（比如"食物类型 = 3"）变成神经网络能用的向量 |

## E · 训练细节

| 术语 | 大白话 |
|---|---|
| **entropy / 熵** | "选择有多随机"的度量。高 = 随机乱选，低 = 死板只选一种 |
| **entropy bonus / 探索奖励** | 训练时鼓励 agent 别太死板，多探索 |
| **advantage / 优势** | "这一步动作比平均水平好多少" |
| **KL divergence / KL 散度** | "新打法和旧打法差多远"的度量。PPO 用它防止一次跨太大 |
| **FP16 / BF16** | 用 16 位表示一个数字（标准是 32 位）。少占一半显存，GPU 算更快，精度略低 |
| **混合精度训练** | 大部分用 FP16 算（快），关键的地方用 FP32（稳） |
| **SIMD** | CPU 一次同时算多个数字的能力（比如一次算 8 个加法） |
| **课程学习** | 先教简单任务，再教难的。和教小孩一样，循序渐进 |
| **POMDP** | "部分可观测"的世界。agent 不能看到全局，只能看到附近一小块 |

## F · 项目专属术语

| 术语 | 大白话 |
|---|---|
| **food / 食物** | 世界里随机撒的小蓝球。agent 撞到就吃了 +10 分。被吃完会自动在另一个随机地方重生 |
| **Perception / 感官** | 把世界状态压成 agent 大脑能吃的"数字串"。S2 是 18 个浮点数：自己速度 3 个 + 最近 5 个食物的相对位置 15 个 |
| **VISION_RANGE** | "感官归一化"用的距离尺度。距离/64m，让数字大致在 [-1,1] 范围里，神经网络更好学 |
| **closest_k** | "找出最近的 k 个食物"的工具函数 |
| **try_eat** | "撞到就吃"的判定函数。返回这一帧吃了几个 |
| **AgentAction** | 一帧里 agent 想做的事（往哪走、吃、喝、攻击、捡、造工具、建屋）。PPO 训练时神经网络就输出这个东西 |
| **wander controller** | 训练前用的占位"打法"——agent 每隔 2 秒随机换方向走。S4 上 PPO 后这玩意被替换 |
| **RewardSystem** | 把"这一帧的世界发生了啥"换算成一个分数（比如吃东西 +1，被狼咬 -1）。PPO 训练时网络追求总分越高越好 |
| **alive_step / 活下去奖励** | 不死就每秒 +0.05 分。鼓励 agent "尽量别死" |
| **vital_pressure / 状态软惩罚** | 饿/渴/累/温度偏离时每秒扣一点分。让 agent "不能不管自己" |
| **death penalty** | 一死扣 100 分。一次死要花 2000 秒"活下去"才能补回来——所以 agent 学到的第一件事是"别死" |
| **EpisodeManager / 一局管理** | 控制"一局"何时开始、何时结束（死或者超时）、结束后怎么重置 agent。地形不变，只重置 agent + 动物 |
| **Observation vector / 观察向量** | 神经网络真正吃的"数字串"。S3 是 124 个数字：自己状态 + 4 棵附近植物 + 4 只动物 + 3 个资源 + 背包 + 庇护所方向 |
| **one-hot 编码** | "这只动物是兔/鹿/狼/鱼/鹰中的哪个"用 5 个 0/1 槽位表示。神经网络喜欢这种格式 |
| **归一化 / normalisation** | 把所有数字都缩到差不多 [-1,1] 范围。神经网络更好学，不被大数字吓跑 |
| **ResourceSystem** | 世界里的"原料"：干柴 / 石头 / 长草。按地形撒，捡了放进背包，过段时间会再生 |
| **Inventory / 背包** | agent 能带的东西。最多 3 格。木棒、石头、长草是基础原料；木+石组合 = 长矛 |
| **craft / 合成** | 把背包里的原料组合成工具。S3 只有"木+石→长矛"一种，后面会扩 |
| **Shelter / 庇护所** | agent 能在地上搭一个简易窝（消耗 1 木 + 1 草）。站在窝里时体温会回归正常，狼也更容易放弃追 |
| **night_factor / 夜行强度** | 0~1 的数字，0=正午，1=深夜。狼在夜间用它放大感知半径（+50%）和速度（+20%）|
| **agent_attack / 主动攻击** | F 键。找前方 90 度扇形里最近的活动物攻击：徒手 1.5m / 20 伤；持长矛 2.5m / 50 伤。打死狼算"狩猎"+5 reward，打死兔/鹿/鱼算吃肉补 hunger |
| **shelter 防狼** | 站在自己造的庇护所内，狼对你的"感知半径"砍到 30%、"追击速度"砍到 70%。蓝图原文"避雨避风避狼" |
| **生态链 / wolf 追猎物** | 狼会自动扫描附近 40m 内的兔/鹿，找到就追，1.5m 内一击秒杀。所以你能看到"狼追兔→兔飞奔逃→狼咬死兔"这样的画面，不只追 agent |
| **mushroom 中毒** | 蘑菇有 30% 概率不补 hunger 反而扣 hunger 12 + stamina 25 + reward 一刀。蓝图原文"小心中毒"。让 agent 学到"蘑菇有风险" |
| **biome 减速 / 加速失水** | swamp 让 agent move_speed 砍半（蓝图"沼泽减速"）。desert 白天让 thirst 减得快 2 倍（蓝图"沙漠快速失水"）|

## H · S3.10 完整生存模拟主线

| 术语 | 大白话 |
|---|---|
| **progression / 主线进度** | "前/中/后期"三阶段。前期一开始啥都没有；中期解锁建房子；后期解锁种田/驯服。蓝图原话"agent 按一条主线走" |
| **解锁条件** | 前→中：采集 30 资源 + 探索 500m；中→后：建好 1 座房子 |
| **课程学习 curriculum learning** | RL 教 AI 的方法：先教简单（采果子），过关再开放下一阶段（建房子）。比一开始扔到大海里强 |
| **稀疏奖励 sparse reward** | 奖励很少出现的设计。比如"建好一座房子 +50"，平时只有小奖励 +0.05/s |
| **EWC / 弹性参数固化** | 防 AI 学新忘旧的技术。学新阶段时给重要旋钮加"摩擦力"，让它们别被改太多 |
| **建造图纸 ConstructionSite** | 中期核心：在地上放"图纸"白框 → agent 来回搬材料堆上去 → 系统每帧数"够没够" → 够了凭空盖好真 3D 房子 |
| **房型** | 4 种：Shed（小棚 4W+2G）/ WoodHouse（木屋 8W+2S+4G）/ StoneHouse（石屋 6W+8S+2G）/ BigHouse（大屋 12W+10S+6G）|
| **房子覆盖（coverage）** | 房子周围一定半径内算"在屋内"，狼 perception/speed 衰减、体温回归 |
| **草裙 / 兽皮护寢** | 衣服。3 草 → 草裙（冷-10%）；2 兽皮 → 兽皮护寢（冷-30%）。Wardrobe 槽位独立，不占背包 |
| **兽皮 Fur** | 杀 rabbit/deer/wolf 时掉落到背包，造兽皮护寢的材料 |
| **种子 Seed** | 摘 fruit/berry 时 30% 概率掉。Late 阶段在 FarmPlot 里种 |
| **农田 FarmPlot** | 后期：N 键种 seed → J 键浇水 → 系统按"水量×时间"长大 → 自动 ripe → E 键收果 |
| **驯服 Tame** | 后期：K 键给动物喂食（rabbit/deer 吃 grass、wolf 吃 fur）→ 累积 trust 到 0.5 → 该动物"following" 跟着 agent 跑 |
| **trust / 信任度** | 0-1，每次喂食 +0.10~0.15。达到 0.5 触发跟随 |
| **解锁 +50 大奖励** | 阶段升级时（Early→Mid 或 Mid→Late）触发的稀疏奖励。引导 PPO "为了通关而努力"|

## H · S4-pre 训练前置 / PPO 框架

| 术语 | 大白话 |
|---|---|
| **headless / 无头模式** | "不开窗口"地跑模拟。训练时不画画面就能比看着画面快几百倍 |
| **Environment 类** | 把"完整一局世界"打包成一个对象。给它一个 action 它告诉你新状态+奖励+是否结束。PPO 只跟它说话，不直接碰世界 |
| **AgentAction 19 通道** | 神经网络这一帧想做的事，包成 19 个数字：3 个连续（移动/转身）+ 16 个 0/1（吃/喝/打/捡/造工具/穿衣/种田/驯服 等）|
| **rollout** | "让 agent 在世界里连续动 N 步"，把这些步全存下来一起学。比"动一步学一步"稳得多 |
| **rollout_steps=2048** | 每次让 agent 连续动 2048 步，够攒一波数据来更新网络 |
| **GAE-λ** | 根据"将来还会拿多少分"算"这一步动作到底好不好"的工具。λ=0.95 表示"看 20 步以内的将来" |
| **clip_eps=0.2** | PPO 安全带：每次更新只允许新打法和旧打法差最多 20%。防止一步学坏 |
| **value head** | 网络的"打分员"分支，估"现在这种局面将来能拿多少分"。GAE 用它做基准对比 |
| **actor head** | 网络的"决策员"分支，输出 19 个动作通道 |
| **log_std** | 连续动作的"乱度"，初值 log(0.5)=-0.69。越大越乱（探索多），越小越准（保守）|
| **MetricsLogger / 指标日志** | 每局 reward 11 分量都写到 .jsonl 文件。统计脚本能算出 PPO 比 wander 好多少 |
| **wander baseline** | 随机策略的下限。PPO 必须打败 wander 平均值 -37 才算"在学"，超过 +50 才算"真懂" |
| **smoke test / 冒烟测试** | 跑一个最小 case 看大流程能不能走通。本身不要求训得好，只要不爆炸 |
| **approx_kl** | "新打法离旧打法多远"。PPO 健康时这个数应该一直很小（<0.05）|
| **clip_frac** | "更新时被 clip 截断的样本比例"。0.1~0.3 算正常，太高说明步子迈太大 |
| **reward hacking / 奖励漏洞** | RL 经典坑：agent 找到奖励函数的 bug 疯狂刷分，看起来 reward 飙升其实啥也没学。我们 ppo_run1 ep39 后水奖励刷到 +6000 就是这个 |
| **reward shaping** | 设计奖励函数让 agent 朝想要的方向学。难点是要"刚好引导"又"不能被钻空子" |
| **actual_delta 接口** | `eat/drink` 返回"真的吸收多少"，奖励绑这个返回值。喝不进去的水不给奖励，是 reward hacking 的解药 |
| **diminishing return / 边际递减** | "越接近满血越少奖励"。actual_delta 接口天然实现这个：thirst 99→100 只能加 1，不会再多 |
| **退化 / regression** | 训练曲线某个分量突然下降。蓝图 line 108"任一退化立刻警报"|

## I · S4.0 记忆架构

| 术语 | 大白话 |
|---|---|
| **GRU / GRUCell** | 一种"有记忆的"神经网络层。它会在脑子里保留一团数字，下一帧融合新感官 + 旧记忆。能记 ~3-10 秒（100-300 步）|
| **hidden state / 隐状态** | GRU 那团记忆数字。每个 episode 开始置零，episode 内一直传下去 |
| **stale-hidden update** | PPO 训练 GRU 时的简化做法：用 rollout 时存的旧 hidden 重新算 loss，不重新展开整个 sequence。CleanRL 这么干 |
| **SpatialMemory / 认知地图** | agent 头脑里挂的一张 64×64 网格，覆盖整个 1024m 地图。走到哪写哪："这格有水"、"这格有狼"。整局不忘 |
| **Neural Map 思路** | Parisotto-Salakhutdinov 2017 论文：用 2D 网格存空间记忆比 DNC 更适合导航。我们 SpatialMemory 就是这个简版 |
| **cell / 格子** | SpatialMemory 的 16m × 16m 单元，每格 6 个 0/1 特征：visited/water/food/resource/danger/building |
| **local window / 局部窗口** | obs 不喂整张 64×64 地图给网络（太大），只读 agent 周围 8×8 格 × 6 特征 = 384 维 |
| **DNC / 可微神经计算机** | 业界一种"超级长期记忆"方案，DeepMind 2016 提的。我们没用——训练太难且场景不合适。详见 Q-019 |
| **MERLIN** | DNC 在 3D 导航的最有名案例（DeepMind 2018）。要 VAE + 复杂 loss，训练贵 5-10x，我们 8GB 显存跑不动 |
| **obs P0 补料** | 把 PPO 看不到的关键信息（水方向/biome/选什么房/图纸缺什么/last_action）加进 obs。先解决"看不到"，再谈记忆 |
| **认知地图清空** | 每个 episode 开始 SpatialMemory.clear()，重新探索。蓝图"世界永不重置但 episode 重置"|

## K · D-041 主线驱动 reward（per-episode milestone）

| 术语 | 大白话 |
|---|---|
| **first_X milestone** | "本局第一次做某件事"的奖励。比如 first_drink = 喝到第一口水 +20。共 11 个：drink/eat/collect/spear/shelter/clothing/hunt/seed_planted/house/crop/follower |
| **per-episode reset** | 每局开始所有 first_X 标志重置，每局都能再拿这 11 个 +20。这是和老设计（once-per-training-run）的关键差别——老设计第 0 局拿完后续局没奖励，新设计每局都有清单 |
| **satiation scale / 饱腹度衰减**（D-041，已被 D-043 drive 取代） | 当前 vital >=90 时再吃/喝 reward = 0；>=80 系数 0.2；>=50 系数 0.5；<50 满奖励 |
| **drive function / 驱力函数** | Keramati 2014 公式 `drive(v) = ((100-v)/100)²`。v=100（饱）→ 0，v=0（空）→ 1。**reward = drive_before - drive_after**。天然包含"饱了就不奖励"，比 satiation_scale 平滑 |
| **drive reduction reward / 驱力差分奖励** | 喝水/吃饭的 reward = 喝前 drive - 喝后 drive。一次完整 0→100 喝水拿 +1 reward，已经 80 喝到 100 只拿 +0.04 reward。**自然抑制饱腹后刷分** |
| **crisis scale / 危机系数** | D-043 P1：max_drive>0.7 时 milestone × 0.3；>0.4 时 × 0.7；否则 × 1.0。让 agent 在饥渴时**先求生再发展**，不会一边渴死一边盖房子拿 milestone |
| **PBRS / 势函数奖励** | Ng/Harada/Russell 1999 ICML：F=γΦ(s')-Φ(s) 形式的 reward shaping。**数学保证不改变最优策略**。问题：持有 Φ>0 时 per-step F 是负的（因 γ<1），不符合"持有状态的持续奖励"。**D-043 P2 没用真 PBRS** |
| **state-based bonus / 状态价值奖励** | D-043 P2：直接 reward += w · Φ(s)。违反 Ng 1999 policy-invariance，但持有状态时奖励持续为正。Φ(s) = 0.5·has_house + 0.3·has_clothes + 0.1·n_followers(cap 0.5) + 0.05·inventory_diversity(cap 0.5) |
| **Φ (phi) state potential / 状态势函数** | 给状态打分的函数。手工设计：累积越多"持有物品/能力"，Φ 越大。max ≈ 1.5 |
| **reward clip / 奖励截断** | D-043 P1：per-step total clip 到 [-10, +10]。防 PPO advantage 估计被 single-step outlier（如多个动作叠加 +20）毁坏 |
| **几何均值 score** | Crafter 论文评估：score = exp(mean(log(1+success_rate_i))) - 1。N 个 milestone 的均衡解锁分数。**广度比深度重要**：解 5 个 100% 比解 1 个 100% 分高 |
| **achievement / 成就 / first_X** | Crafter 概念：每 episode 可以"解锁一次"的语义事件。本项目用 first_X（drink/eat/collect/spear/shelter/clothing/seed/house/crop/follower，共 10 个，D-043 删了 first_hunt） |
| **specification gaming / 规约游戏** | DeepMind 概念：agent 字面满足 reward 函数但违反设计意图。例：撩狼为 first_hunt = 教科书例子（D-043 之前的 bug） |
| **主线 / main line** | 蓝图原话"agent 按一条主线走"。RL 工程实现 = 11 个 first_X 串成的"今天该做的事"清单，覆盖求生→工具→建造→农耕/饲养全流程 |
| **过日子链条** | 用户原话。指 agent 行为应该像在过日子，前期 → 中期 → 后期渐进，不是 3 个按键孤立触发。Phase 2 会扩展为：找种子 → 造工具 → 耕地 → 播种 → 浇水 → 收获 → 存储 |
| **方向感 vs 无头苍蝇** | 用户洞察：agent 行为允许"无头苍蝇一点"（自然性），但**整体偏向主线**。RL 工程实现 = 主线 reward 总量 > 单一密集 reward |
| **fire_once** | Environment.cpp 的辅助 lambda：`if (!flag) { flag = true; reward.event_milestone(); }`。一行触发一个 milestone |

## J · D-039 多项化策略（multinomial）

| 术语 | 大白话 |
|---|---|
| **Bernoulli 头** | 旧设计：16 个独立的 0/1 开关，每个开关有自己的概率 p_i。一步可以同时按下多个键 |
| **Categorical 头 / multinomial 头** | 新设计：17 路 softmax 概率分布，一步**只**选一个动作（或 NOOP）。强制互斥 |
| **softmax** | 把一堆数 x_i 变成"和为 1"的概率：p_i = e^{x_i} / Σ e^{x_j}。"哪个数最大对应概率最大" |
| **NOOP / 空操作** | "这步什么都不做"。Categorical 第 17 类。让 agent 能耐心等待条件成熟，不是被强迫每步出招 |
| **熵塌缩 / entropy collapse** | 策略学不出条件→所有动作概率都接近均匀（高熵）→PPO 卡在"瞎按一通"。D-039 之前的 ppo_run3 就是这病 |
| **policy collapse** | 概率分布退化到一个动作上（熵≈0）。完全没探索。坏。D-039 期望训练曲线 entropy 从 5 慢慢降到 1-2，不是一冲到 0 |
| **action histogram / 行为直方图** | 一局里每种动作被选了多少次的统计。诊断"策略在干嘛"的核心工具，scripts/analyze_actions.py 算 |
| **诊断驱动** | 不要凭直觉调参。先抓数据看 agent 真实在做什么，再决定改什么。D-039 就是这么定位到"多 sigmoid 全 0.5"这个 root cause |

## 天气 / weather（D-121）
- **天气系统**：天上会变天，晴天和下雨来回切。下雨时：露天的火会被浇灭（有屋顶 / 沙漠不会）、身上会淋湿、农田和喝水会自动补上。默认关着，命令加 `--weather` 才开。
- **湿度 wetness**：身上有多湿，0=干 1=湿透。淋雨升高、躲屋里慢慢变干。湿着过夜掉体温更快（更容易冻死）。
- **雨强 rain_intensity**：天上雨下得多大，0=没雨 1=最大。
- **新血脉 / obs 手术**：给 AI 的"感官"清单末尾加新格子（这次加了雨强、湿度 2 个）。老格子顺序一个不动，新格子先填 0，老模型直接能接着用、不用从头学。

## D-122 新增
- **一锅炖（I/O 锁死）**：把网络的"输入口/输出口"数量一次性预留满（含以后才用的机制），以后加新机制只在世界端开关、永不再改网络尺寸——这样老存档永远能续用、不用重头练。
- **热启动手术**：老存档装进更大的新网络时，把新增的那几行/列权重全设成 0。效果：第一天输出和老网络一模一样（0×真值=0），行为零扰动；新口再慢慢从 0 学出来。
- **烤肉(cook)**：原始人站在点着的火旁、手里有生肉时，烧掉一点火的燃料，把 1 块生肉变成 1 块熟肉。熟肉更顶饿、蛋白更高，但要先打猎拿到生肉、还要一直喂火——不是白送。
- **照明(light)**：夜里野果本来采不到；但站在自己点着的火旁就有光，能继续采集。光来自"自己生的火"，有代价（要烧木头）。
- **里程碑(milestone)**：某件事"第一次做到"时一次性给的奖励（如第一次喝水/第一次烤肉），引导它去尝试新行为；只发一次，防止反复刷分。

## D-122 RUNG 2-5 新增
- **挖矿(mine)**：原始人得先钻进山洞、走到矿点旁、花体力才能挖到矿石——不在洞里按了也没用。这给了山洞一个"非进不可"的理由。
- **石斧/石镐(石器)**：用 1 块木 + 1 块矿造出来的工具。石斧砍树更快、石镐挖矿更省力，是"先攒矿造工具"这条绕路的回报。
- **建造升级门(tier-gate)**：好房子要好工具才能盖——石头屋要石镐、大屋要斧+镐；草棚/木屋永远免费。逼它沿"工具→更好的家"往上爬。
- **巨炉/纪念碑(monument)**：一座要一趟趟搬矿石、搬满 10 趟才建成的大工程。建成后周围会散发暖意(取暖光环)。
- **Boss(定时寒潮)**：每条命到固定时间(第180秒)必来的一场极寒+猛兽考验，跟你建没建巨炉无关。巨炉是"护盾"——赶在 Boss 来之前建好，它的暖意光环就能帮你扛过寒潮；没建好只能靠火+躲屋里硬扛。活过整场=通关、给双倍奖励。

## D-122 审计相关
- **机制闭环（值不值得做）**：站原始人视角，一条科技链要"值得爬"得满足三条——前置拿得到、收益>成本、不爬会出事(有压力)。三条缺一就没人做。审计发现整套机制"能做但没理由做"。
- **基准线吊床**：躺着摘果子+窝棚回血就能永生→所有科技都是白费劲。修法=造稀缺(寒夜不回饥饿)，不是加胡萝卜。
- **精确信用分配/溯源**：强化学习核心难题"凭啥知道是哪步害死的"。死了扣分会被 GAE 沿决策链往回"打折抹开"成一团雾，不精确。解法=给每个指标加"提前预警的连续凸惩罚"，快没水就连续扣，锅落在放任那个指标的那段路上。
- **凸惩罚（squared/平方）**：亏空取平方→单个垮掉的指标罚得远比"几个都半垮"重，逼网络盯住那个真正快归零的指标，而不是平摊。

## D-122 提前预警 + SF
- **advantage(优势)**：PPO 给每个动作算的一个分——"这一步比平均水平好(正)还是差(负)"。好→下次更可能做，差→下次更少做。这是 PPO 唯一的前进方向。
- **提前预警仪表盘**：训练日志每行末尾的 `hunt_adv/cook_adv` ——盯"打猎/烤肉动作 advantage 的符号"，正=该行为会单调涨(有戏)、负=会被掐灭(白跑)、n=0=压根没点火。几百回合判趋势，不用跑满 4800 万步。
- **Successor Features(SF/继承特征)**：训一次学出 ψ="从此刻起未来会累计多少每一种奖励分项"。之后任何新的奖励权重 w′，用 `ψ·w′` 当场估出"这套配方值多少分"，不重训。最像你说的"花一份算力试多种配方"。前提：奖励是各分项的线性加权和(我们正好是)。设计稿见 docs/Successor_Features_设计.md。
- **量子叠加跑多结果？**：你设想的"一张图多主角、共享资源却互不感知"是自相矛盾的(共享=偷偷干扰、真独立=等于多张图没省算力)。真正省的是 GPU 批量算(已在做)。能用的：①并行多环境(已用，`_e`/`_c2` A/B 就是)②群体训练 PBT ③SF(最接近)。
